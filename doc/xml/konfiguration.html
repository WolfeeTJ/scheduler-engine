Konfiguration

Beim Start des Scheduler geben Sie mit dem Parameter -config= die Datei an, die die in XML formulierte Konfiguration enthält.

Beispiel für eine einfache Konfigurationsdatei mit einem Job in JavaScript (für Windows):

<?xml version="1.0" encoding="iso-8859-1"?>
<spooler>
    <config>
        <process_classes/>
        <jobs>
            <job name="test_job" title="Das ist der Titel des Jobs test_job">
                <script language="JavaScript"><![CDATA[

function spooler_process()
{
    spooler_log.info( "Hallo Welt!" );
    return false;
}
                ]]></script>

                <run_time repeat="10"/>
            </job> </jobs>
    </config>
</spooler>


Konfiguriert ist hier der Job test_job, den der Scheduler alle 10 Sekunden wiederholen wird. Wenn diese Konfiguration 
in der Datei scheduler.xml abgelegt ist, können sie den Scheduler so starten:


scheduler -config=scheduler.xml -log-dir=*stderr
Der Parameter -log-dir=*stderr stellt ein, dass der Scheduler das 
Protokoll nach stderr, also zum Schirm schreibt. Mit Ctrl-C lässt sich der Scheduler abbbrechen.

<!ELEMENT spooler ( config+ | command | answer )>

<!ELEMENT config ( security?, object_set_classes?, threads? )>

Enthält die vollständige Konfiguration für einen SPOOLER.

<!ATTLIST config spooler_id CDATA ''>

Der SPOOLER wählt die Konfiguration aus, deren Spooler-Id leer ist oder mit der beim Start des SPOOLERS angegebenen 
übereinstimmt. Eine passende Konfiguration muss vorhanden sein. Nachfolgende Konfigurationen werden nicht beachtet. Eine 
Default-Konfiguration mit spooler_id="" sollte daher zum Schluss angegeben werden.

<!ATTLIST config log_dir CDATA #IMPLIED>

In dieses Verzeichnis werden die SPOOLER-Logs geschrieben. Wenn *stderr angegeben ist, wird das Log nach stderr 
geschrieben. 

Wenn die Log-Datei in dem Verzeichnis nicht angelegt werden kann, beendet sich der SPOOLER.

<!ATTLIST config tcp_port CDATA '4444'>

TCP-Port, an dem der SPOOLER Kommandos entgegen nimmt.

Wenn der Port nicht geöffnet werden kann, beendet sich der SPOOLER.

Bei tcp_port="0" wird kein TCP-Port geöffnet, dadurch kann die Steuerung des SPOOLERS per TCP-Kommando, z.B. über eine 
Web-Seite, unterbunden werden.

<!ATTLIST config udp_port CDATA '4444'>

UDP-Port, an dem der SPOOLER Kommandos entgegen nimmt.

Wenn der Port nicht geöffnet werden kann, beendet sich der SPOOLER.

Bei udp_port="0" wird kein UDP-Port geöffnet, dadurch kann die Steuerung des SPOOLERS per TCP-Kommando, z.B. über eine 
Web-Seite, unterbunden werden.

<!ATTLIST config priority_max CDATA '1000'>

Höchste Priorität, die eine Task haben kann.

<!ATTLIST config param CDATA ''>

Parameter, der den Skripten als Eigenschaft spooler.param bereitgestellt wird. 

<!ATTLIST config include_path CDATA ''>

Verzeichnis, das auf relative Dateinamen im Element <include> (in <script>) angewendet werden soll.

<!ATTLIST config free_threading CDATA 'no'>

Bei free_threading='no' (default) werden die Threads serialisiert. Bei free_threading='yes' laufen die Threads parallel. 
Dabei ist auf Wechselwirkungen zu achten, die bei der Serialisierung nicht auftreten. Im Element <thread> kann einzelnen 
Threads die Eigenschaft gegeben werden, parallel zu den anderen zu laufen. 



<!ELEMENT security ( allowed_host* )>

Setzt die Sicherheitseinstellungen.

<!ATTLIST security ignore_unknown_hosts (yes|no) 'no'>

Wenn im Element allowed_host ein unbekannter bzw. nicht auflösbarer Hostname angegeben wird, wird der Start des SPOOLERS 
trotzdem fortgesetzt.

<!ELEMENT allowed_host EMPTY>

Gibt einen Host oder ein Netzwerk an, die mit dem SPOOLER über TCP oder UDP kommunizieren dürfen. Ohne diesem Element 
ist die Kommunikation über TCP und UDP gesperrt.

Eine häufig anzutreffende Einstellung ist localhost bzw. 127.0.0.1, wenn der Server, auf dem der SPOOLER abläuft 
gleichzeitig den Web-Server mit der Web-Seite zur SPOOLER-Kontrolle betreibt; die Einstellung verhindert, dass von 
anderen Rechnern mit dem SPOOLER kommuniziert werden kann. 

<!ATTLIST allowed_host host CDATA #REQUIRED>

Name oder IP-Nummer des Hosts oder die IP-Nummer eines Netzwerks der Klasse A, B oder C. Der Name eines Netzwerks kann 
nicht angegeben werden.

Die Einstellung für einen Host hat Vorrang vor der Einstellung für ein Netzwerk.

<!ATTLIST allowed_host level (none|signal|info|all) #REQUIRED>

level="none": Nichts ist zugelassen.

level="signal": Nur <signal-object> über UDP ist zugelassen.

level="info": <show_state> ist zugelassen.

level="all": Alle Kommandos sind zugelassen.



<!ELEMENT object_set_classes ( object_set_class+ )>

Enthält die Objektmengenklassen.



<!ELEMENT object_set_class ( level_decls?, script? )>

Eine Objektmengenklasse ist beschrieben durch die bezeichnete Stufe der Objekte und ein Skript, das die Methoden für 
eine Objektmenge und deren Objekte enthält.

<!ATTLIST object_set_class name CDATA #REQUIRED>

Name der Klasse.

Beispiel: Dokumente

<!ATTLIST object_set_class script_interface (oo|procedural) 'oo'> 

Gibt an, ob die objektorientierte oder die prozedurale Schnittstelle verwendet werden soll.



<!ELEMENT level_decls ( level_decl+ )>

Verzeichnis der Stufen, die ein Objekt haben kann. Das Verzeichnis muss nicht vollständig sein und kann auch ganz 
fehlen.



<!ELEMENT level_decl EMPTY>

Jedes Objekt hat eine Stufe, die durch eine Nummer angegeben ist. Den Stufennummern werden hier Bezeichnungen 
zugeordnet.

<!ATTLIST level_decl level CDATA #REQUIRED>

Nummer der Stufe. Die Nummer ist eine natürliche Zahl. Eine höhere Nummer bezeichnet eine höhere Stufe.

<!ATTLIST level_decl name CDATA #REQUIRED>

Bezeichnung der Stufe.



<!ELEMENT script ( #PCDATA | include )*>

Skript mit den Methoden für eine Objektmenge und deren Objekte.

<!ATTLIST script language CDATA 'VBScript'>

Jede installierte Skriptsprache, die nach Microsofts Scripting Engines implementiert ist, kann angegeben werden.

Bespiele: VBScript, JScript, PerlScript



<!ATTLIST script use_engine (task|job) 'task'>

Mit use_engine="task" erhält jede Task eine eigene Scripting Engine. Damit sind die Tasks eines Jobs voneinander 
isoliert.

Mit use_engine="job" wird dieselbe Scripting Engine für alle Tasks desselben Jobs verwendet. Die Scripting Engine wird 
nur beendet, wenn die Task gestoppt wird. Die Routine spooler_init() des Skripts wird nur einmal vor der ersten Task 
gerufen, spooler_exit() nur einmal nach der letzten Task.

Für Skripte mit der Sprache PerlScript sollte die Einstellung use_engine="task" gewählt werden.

Zum Aufbau des Skripts s.u.

<!ELEMENT include EMPTY>

Fügt an der Stelle Text aus einer anderen Datei ein. Damit kann ein Skript eine mehrfach verwendbare Datei nutzen.

<!ATTLIST include file CDATA #REQUIRED>

Gibt den Namen der Datei an, deren Inhalt hier eingesetzt werden soll. Wenn der Dateiname relativ ist, wird das 
Verzeichnis aus include_path verwendet.



<!ELEMENT threads ( thread+ )>

Alle Threads werden hier aufgeführt. Damit der SPOOLER überhaupt einen Job ausführen kann, muss ein Thread definiert 
werden. In der Regel genügt ein Thread. Werden mehrere Threads definiert, sollte auf Komplikationen geachtet werden, die 
vielleicht durch die Nebenläufigkeit auftreten können.



<!ELEMENT thread ( script?, jobs? )>

Ein Thread hat ein Thread-Skript und Jobs. Das Thread-Skript wird mit dem Thread gestartet und bleibt solange aktiv bis 
der Thread beendet wird (in der Regel erst bei Beendigung des SPOOLERS). Die Skripte der Jobs können auf das 
Thread-Skript zugreifen über spooler_thread.script.

<!ATTLIST thread spooler_id CDATA ''>

Kennung des SPOOLERS, für den der Thread bestimmt ist. Der Thread wird vom SPOOLER nur berücksichtigt, wenn dieses 
Attribut nicht leer ist und mit der Option -spooler-id= vom SPOOLER-Start übereinstimmt. 

<!ATTLIST thread include_path CDATA #IMPLIED>

Gibt das Verzeichnis für relative Dateinamen im <include>-Element für Skripte dieses Threads an.

<!ATTLIST thread free_threading CDATA #IMPLIED> 

Default ist <config free_threading="no">. Mit free_threading="yes" wird dieser Thread nebenläufig geschaltet.

<!ATTLIST thread priority CDATA #IMPLIED>

Gibt die relative Priorität des Threads im Betriebssystem Windows an. Werte von -15 (niedriger) bis +2 (höher) können 
angegeben werden. 0 ist neutral. "idle" gibt die niedrigste Priorität an



<!ELEMENT jobs ( job+ )>

Alle Jobs werden hier aufgeführt.



<!ELEMENT job ( ( object_set | script | process ), run_time? )>

Ein Job beschreibt eine Aufgabe, die zu bestimmten Zeiten durchzuführen ist. Ein Job kann sich auf eine Objektmenge 
beziehen oder ein eigenes Skript haben. 

Gelangt ein Job zur Ausführung, entsteht eine Task. Von einem Job kann nur eine Task laufen.

<!ATTLIST job name CDATA #REQUIRED>

Name, der den Job identifiziert.

<!ATTLIST job priority CDATA '1'>

Eine höhere Zahl bezeichnet eine höhere Priorität. Niedrigste Priorität ist 0, die höchste ist im Attribut max-priority 
im Element config festgelegt.

Die niedrigste und die höchste Priorität haben besondere Bedeutungen: Priorität 0 lässt eine Task nur ausgeführen, wenn 
kein keine Task mit höherer Priorität läuft, wenn also sonst nichts zu tun ist. Die höchste Priorität gibt absoluten 
Vorrang. Tasks mit niedriger Priorität werden solange angehalten, bis keine Task mit der höchsten Priorität mehr läuft.

<!ATTLIST job output_level CDATA #IMPLIED>

Wenn der Job mit einer Objektmenge arbeitet, bezeichnet dieses Attribut die Stufe, auf die der Job die Objekte der 
Objektmenge befördert. Dieses Attribut muss in diesem Fall angegeben werden.

<!ATTLIST job spooler_id CDATA ''>

Kennung des SPOOLERS, für den der Job bestimmt ist. Der Job wird vom SPOOLER nur berücksichtigt, wenn dieses Attribut 
nicht leer ist und mit der Option -spooler-id= vom SPOOLER-Start übereinstimmt. 



<!ELEMENT object_set ( levels? )>

Beschreibt die Objektmenge, mit der der Job arbeitet.

<!ATTLIST object_set class CDATA #REQUIRED>

Name der Objektmengenklasse, wie sie mit einem Element object_set_class deklariert ist.



<!ELEMENT levels EMPTY> 

Intervall von Stufen, auf die die Objektmenge beschränkt werden soll.

<!ATTLIST levels low CDATA #REQUIRED>

Kleinste Stufennummer.

<!ATTLIST levels high CDATA #REQUIRED>

Nummer der Stufe, die nicht mehr berücksichtigt werden soll.

<!ELEMENT process EMPTY>

Beim Start des Jobs, also zu Beginn einer Task, wird ein Prozess eingerichtet. Der Prozess ist nebenläufig zum Thread. 
Der Spooler überwacht den Prozess und beendet die Task nachdem der Prozess sich beendet hat. Ein exit code verschieden 
zu 0 wird als Fehler interpretiert. Der exit code ist das Ergebnis der Task (in task.result).

<!ATTLIST process file CDATA #REQUIRED>

Name der Datei, die das Programm oder das Skript enthält, mit dem der Prozess gestartet werden soll.

<!ATTLIST process param CDATA ''>

Gibt den Parameterstring an, der dem Prozess übergeben wird.

<!ELEMENT run_time ( period*, date*, weekdays?, monthdays?, ultimos?, holiday* )>

Gibt die Zeiten an, zu denen der Job laufen soll. Wenn das Element run_time fehlt, wird der Job nicht nach Zeit 
gestartet.

Ein Job kann einmal zu einer bestimmten Uhrzeit (single_start) oder mehrmals in einem Zeitraum (begin) gestartet werden.

<!ATTLIST run_time single_start CDATA #IMPLIED>

Voreinstellung für Attribut begin des Elements period (Tageszeit, zu der der Job einmal gestartet werden soll).

Format: HH:MM[:SS]

Die Zeiten von 00:00 bis 23:59:59 können angegeben werden.

Die Task wird zur end_time beendet, wenn nicht let_run="yes" ist.

<!ATTLIST run_time begin CDATA '00:00:00'>

Voreinstellung für Attribut begin des Elements period (Tageszeit, ab der die Task laufen soll).

Format: HH:MM[:SS]

begin kann nicht mit single_start kombiniert werden.

<!ATTLIST run_time end CDATA '24:00:00'>

Voreinstellung für Attribut end des Elements period (Tageszeit, bis vor der die Task laufen soll). 

Format: HH:MM[:SS]

<!ATTLIST run_time let_run (yes|no) 'no'>

Voreinstellung für Attribut let_run des Elements period (gibt an, ob beim Erreichen der Endezeit kein Schritt mehr 
ausgeführt und die Task beendet werden soll).

<!ATTLIST run_time repeat CDATA '5'>

Voreinstellung für Attribut repeat des Elements period (gibt die Zeit in Sekunden an, die längstens zwischen dem Ende 
einer Task und dem erneuten Start verstreichen soll).

Kann nicht mit single_start kominiert werden.



<!ELEMENT date period>

Gibt ein Tag an, an dem der Job laufen soll.

<!ATTLIST date date CDATA #IMPLIED>

Datum des Tags im Format yyyy-mm-dd (oder ein anderes hostWare-Format).



<!ELEMENT weekdays ( day+ )>

Gibt die Wochentage an, an denen der Job laufen soll. 0 bezeichnet den Sonntag, 1 Montag usw. bis 6 Sonnabend.



<!ELEMENT monthdays ( day+ )>

Gibt die Tage im Monat an, an denen der Job laufen soll.



<!ELEMENT ultimos ( day+ )>

Gibt die Tage im Monat an, rückwärts vom letzen Tag gezählt, an denen der Job laufen soll. 0 bezeichnet den letzen Tag, 
1 den vorletzten usw.



<!ELEMENT holiday EMPTY> 

Gibt die Tage an, an denen der Job nicht starten soll. Vorher eingestellte Tage, auch mit dem Element date, werden mit 
dieser Angabe widerrufen.

<!ATTLIST holiday date CDATA #REQUIRED>

Datum des Tags im Format yyyy-mm-dd (oder ein anderes hostWare-Format).



<!ELEMENT day period>

<!ATTLIST day day CDATA #REQUIRED>

Gibt für die Elemente weekdays, monthdays und ultimos die Nummern der Tage an.



<!ELEMENT period EMPTY>

Beschreibt die Tageszeit, zu der der Job laufen soll. 

<!ATTLIST period single_start CDATA #IMPLIED> 

Tageszeit, zu der der Job einmal gestartet werden soll.

Format: HH:MM[:SS]

Die Zeiten von 00:00 bis 23:59:59 können angegeben werden.

Die Task wird zur end_time beendet, wenn nicht let_run="yes" ist.

Kann nicht zusammen mit den Attributen begin und repeat angegeben werden.

Voreinstellung ist die im Element run_time gemachte Angabe.



<!ATTLIST period begin CDATA #IMPLIED> 

Tageszeit, ab der die Task laufen soll.

Format: HH:MM[:SS]

Die Zeiten von 00:00 bis 23:59:59 können angegeben werden.

begin kann nicht mit dem Attribut single_start kombiniert werden.

Voreinstellung ist die im Element run_time gemachte Angabe.



<!ATTLIST period end CDATA #IMPLIED> 

Tageszeit, bis zu der die Task laufen soll. 

Format: HH:MM[:SS]

Die Zeiten von 00:00 bis 24:00:00 können angegeben werden.

Wenn Beginn- und Endezeit übereinstimmen, wird bei let_run=no die Task zur angegeben Zeit gestartet und nur ein Schritt 
ausgeführt.

Voreinstellung ist die im Element run_time gemachte Angabe.



<!ATTLIST period let_run (yes|no) #IMPLIED> 

Gibt an, ob beim Erreichen der Endezeit kein Schritt mehr ausgeführt und die Task beendet werden soll 

Voreinstellung ist die im Element run_time gemachte Angabe.



<!ATTLIST period repeat CDATA #IMPLIED>

Gibt die Zeit in Sekunden an, die längstens zwischen dem Ende einer Task und dem erneuten Start verstreichen soll).

Kann nicht mit dem Attribut single_start kominiert werden.

Voreinstellung ist die im Element run_time gemachte Angabe.
