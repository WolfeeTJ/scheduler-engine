Aufträge im Spooler - Klasse Order_queue

18. September 2002

Eigenschaften und Methoden

length
add_order()

length

Liefert die Anzahl der Aufträge in der Auftragsliste.

add_order()

Fügt einen Auftrag der Auftragsliste hinzu.

 

order_queue.add_order( order_or_payload  )


order_or_payload

Wenn der Parameter kein Auftrag ist, erzeugt der Aufruf einen und setzt dessen Eigenschaft payload auf diesen Parameter. 

Der Auftrag wird der Jobkette hinzugefügt. 

Wenn die Eigenschaft state des Auftrags gesetzt ist, wird der Auftrag an die passende Stelle in der Jobkette gesetzt. Die Jobkette muss den Zustand definiert haben (mit add_job()).

Wenn der Auftrag bereits in einer Jobkette oder Auftragsliste ist, wird er zuvor daraus entfernt.

Wenn die id des Auftrags nicht gesetzt ist, erzeugt der Spooler eine (nämlich eine fortlaufende Nummer). 

Wenn die Auftragsliste bereits eine Auftrag mit derselben id hat, wird dieser ersetzt.


Rückgabe

Der Aufruf liefert als Rückgabe den Auftrag.

Hinweis zur Geschwindigkeit

Wenn ein Auftrag in der Auftragsliste eine selbst gesetzte Id (order.id) oder Priorität (order.priority) hat, sollte die Auftragsliste nur mit einigen Hundert Aufträgen gefüllt werden, weil darüber hinaus add_order() langsam wird. Die Laufzeit steigt quadratisch mit der Länge der Auftragsliste.
