Typ State  (Zustand)

Jeder Auftrag hat einen Zustand. Aus dem Zustand geht die Position des Auftrags in der Jobkette hervor. In einer Jobkette ist jedem Zustand ein Job zugeordnet.

Der Zustand ist ein Variant, der sich als Text darstellen lassen muss. Großschreibung ist relevant. Überlicherweise werden die Typen Integer oder String verwendet. 

Typ Priorität

Jeder Auftrag hat eine Priorität. Sie ist eine ganze Zahl, voreingestellt ist 0. Ein höherer Wert bedeutet eine höhere Priorität. Ein Auftrag mit hoher Priorität wird vor Aufträgen mit niederer Priorität in derselben Auftragsliste abgearbeitet.

Typ Payload

Jeder Auftrag hat eine Nutzlast mit den Daten des Auftrags. Das ist ein Variant, der einen beliebigen Wert enthalten kann. Der Wert wird vom Spooler nicht angesehen.
