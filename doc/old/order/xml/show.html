Attribut what - Welche Details sollen gezeigt werden

Den Kommandos <show_state>, <show_job> und <show_job_chains> kann das Attribut what hinzugefügt werden. Angegeben werden kann eine Liste (durch Komma getrennt) von folgenden Schlüsselwörtern: all, task_queue, orders, description, log und standard.

 

 
what="task_queue" 		  
Zeigt im Element <task> die Liste der wartenden Tasks
 
what="orders" 		  
Zeigt im Element <task> die Liste der wartenden Aufträge
 
what="description" 		  
Zeigt das Element <description>
 
what="log" 		  
Zeigt bei <show_history> das Log
 
what="standard" 		  
Zeigt das Minimum (Voreinstellung)
 
what="all" 
 
<show_state> :		  
wie what="description,task_queue"
<show_job> : wie what="description,task_queue"
<show_job_chains> : wie what="description,orders"

Kommando  <show_state>  -  Spooler-Zustand zeigen

<show_state what="..."/>

Mit dem Attribut what kann die Liste der Aufträge gefordert werden: what="orders".

Antwort-Element ist <state>.

Kommando  <show_job>  -  Spooler-Zustand zeigen

<show_job what="..."/>

Mit dem Attribut what kann die Liste der Aufträge gefordert werden: what="orders".

Antwort-Element ist <task>.

Kommando  <show_job_chains>  -  Jobketten zeigen

<show_job_chains what="..."/>

Mit dem Attribut what kann die Liste der Aufträge gefordert werden: what="orders". Nur Aufträge dieser Jobkette werden gezeigt. Aufträge aus anderen Jobketten werden nicht gezeigt (ein Job kann in mehreren Jobketten sein, hat aber nur eine Auftragsliste).

Antwort-Element ist <job_chains>.

Kommando  <show_order>  -  Auftrag zeigen

<show_order job_chain="jobkette" order="id"/>

Die Jobkette muss angegeben werden.

Antwort-Element ist <order>.

Antwort-Element  <job_chains>

Attribute

name="jobkettenname"

Elemente

Eine Liste von <job_chain>.

Antwort-Element  <job_chain>

Attribute

state="state"

next_state="state"

Falls gesetzt.

error_state="state"

Falls gesetzt.

Elemente

<task>

Antwort-Element  <task>

Attribute

in_process_since="yyyy-mm-dd hh:mm:ss.mmmm"

Gesetzt, wenn der Auftrag gerade in Verarbeitung ist. Gibt den Zeitpunkt des Aufrufs von spooler_process() an.

state="..."

Der neue Zustand running_waiting_for_order gilt für eine laufende auftragsgesteuerte Task, die mangels Aufträge angehalten ist.

Elemente

<order>, wenn der Job gerade einen Auftrag ausführt.

<order_queue> bei einem auftragsgesteuerten Job.

Antwort-Element  <order_queue>

Attribute

length="auftragsanzahl"

Elemente

Eine Liste von <order>, wenn mit what="orders" bestellt.

Antwort-Element  <order>

Attribute

id="id"

title="title"

state="state"

job_chain="jobkette"

Gesetzt, wenn der Auftrag in einer Jobkette ist.

job="job"

Der Job, der den Auftrag als nächstes ausführt oder gerade ausführt.

in_process_since="yyyy-mm-dd hh:mm:ss.mmmm"

Gesetzt, wenn der Auftrag gerade in Verarbeitung ist. Gibt den Zeitpunkt des Aufrufs von spooler_process() an.

state_text="statetext"

priority="priorität"
